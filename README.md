# Search Engine Tool - COMP3011 Coursework 2

A Python-based web search tool that crawls websites, builds inverted indices, and enables fast full-text search. Designed for educational purposes to understand search engine fundamentals.

## Project Overview

This search tool implements the core components of a search engine:

1. **Web Crawler** - Respects politeness windows and extracts content from web pages
2. **Inverted Index** - Efficient data structure mapping words to their page occurrences
3. **Search Engine** - Enables single-word and multi-word searches with AND logic
4. **Persistence Layer** - Saves and loads indices for reuse

### Features

- ✅ Concurrent web crawling with enforced politeness window (6+ seconds between requests)
- ✅ Case-insensitive full-text indexing
- ✅ Inverted index with word statistics (frequency, positions)
- ✅ Single-word and multi-word (AND) search queries
- ✅ Phrase search using quotes (e.g., `"good friends"`)
- ✅ Boolean queries with `AND`, `OR`, `NOT`, and parentheses
- ✅ TF-IDF ranked retrieval via `rank` command
- ✅ Persistent index storage to JSON file
- ✅ Interactive command-line interface
- ✅ Comprehensive error handling and logging
- ✅ 68 unit/performance tests with broad coverage
- ✅ GitHub Actions CI pipeline across Python 3.10-3.12

## Usage

### Running the Tool

Start the interactive shell:

```bash
python3 src/main.py
```

### Commands

#### `build`
Crawls the target website and builds the inverted index.

```
> build
```

- Crawls all pages on `https://quotes.toscrape.com/`
- Respects 6-second politeness window between requests
- Creates inverted index with word statistics
- Saves index to `data/index.json`
- **Duration**: 5-10 minutes depending on site size

**Example Output:**
```
Building index...
Crawling https://quotes.toscrape.com/
This may take several minutes due to politeness window.

Crawled 50 pages

Index Statistics:
  Unique Words: 1,234
  Indexed Pages: 50
  Total Word Occurrences: 15,678

✓ Index built and saved successfully!
```

#### `load`
Loads a previously built index from disk.

```
> load
```

- Loads index from `data/index.json`
- Must run after `build` has been executed at least once
- **Duration**: Instant (< 1 second)

**Example Output:**
```
Loading index...
✓ Index loaded successfully!
  Unique Words: 1,234
  Indexed Pages: 50
```

#### `print <word>`
Displays detailed index information for a specific word.

```
> print nonsense
```

- Shows all pages containing the word
- Displays word frequency per page
- Results sorted by frequency (most frequent first)
- Case-insensitive search

**Example Output:**
```
Word: nonsense
Total Occurrences: 5
Pages Containing Word: 2

Pages:
  - https://quotes.toscrape.com/page/1/
    Frequency: 3
  - https://quotes.toscrape.com/page/2/
    Frequency: 2
```

#### `find <word1> [word2 word3...]`
Searches for pages containing all specified words (AND query).

```
> find good friends
```

- Single word: Returns all pages containing that word
- Multiple words: Returns pages containing ALL words
- Phrase query: Use quotes, e.g. `find "good friends"`
- Boolean query: Use operators, e.g. `find life AND NOT death`
- Case-insensitive search
- Results sorted alphabetically by URL

**Examples:**
```
> find indifference
Results for query: 'indifference' (3 pages):

1. https://quotes.toscrape.com/page/1/
2. https://quotes.toscrape.com/page/2/
3. https://quotes.toscrape.com/page/4/

> find good friends
Results for query: 'good friends' (1 page):

1. https://quotes.toscrape.com/page/3/

> find "good friends" OR life
Results for query: '"good friends" OR life' (... pages)
```

#### `rank <query>`
Returns TF-IDF ranked results.

```
> rank "good friends" OR life
```

- Applies boolean/phrase filtering first
- Ranks candidate pages by aggregated TF-IDF score
- Returns highest-relevance pages first

#### `help`
Displays available commands and usage information.

#### `quit` or `exit`
Exits the program.

### Core Components

#### WebCrawler (`src/crawler.py`)

**Responsibilities:**
- Fetches web pages from target domain
- Extracts text content from HTML
- Identifies and queues links for crawling
- Enforces politeness window (6 seconds minimum between requests)
- Tracks visited URLs to avoid duplicates
- Handles network errors gracefully

**Key Methods:**
- `crawl()` - Main crawling loop
- `fetch_page()` - Respects politeness window and fetches single page
- `_extract_text_from_page()` - Strips HTML and returns text
- `_get_next_urls()` - Extracts links from HTML

**Example Usage:**
```python
from src.crawler import WebCrawler

crawler = WebCrawler("https://quotes.toscrape.com/", politeness_window=6)
pages = crawler.crawl(max_pages=100)
```

#### InvertedIndex (`src/indexer.py`)

**Responsibilities:**
- Tokenizes text (lowercase, removes punctuation)
- Creates inverted index mapping words to pages
- Tracks word statistics (frequency, positions)
- Supports multi-word AND queries
- Generates index statistics

**Data Structure:**
```python
{
    'word': {
        'pages': [
            {
                'url': 'https://example.com/page1',
                'frequency': 3,
                'positions': [0, 45, 127, ...]
            },
            ...
        ]
    },
    ...
}
```

**Key Methods:**
- `index_page()` - Add single page to index
- `build_from_pages()` - Build index from page dictionary
- `search_single_word()` - Find pages with word
- `search_all_words()` - AND query for multiple words
- `get_statistics()` - Index statistics

**Example Usage:**
```python
from src.indexer import InvertedIndex

index = InvertedIndex()
index.build_from_pages({'https://example.com/1': 'hello world'})
results = index.search_single_word('hello')
```

#### SearchEngine (`src/search.py`)

**Responsibilities:**
- Processes user search queries
- Formats search results for display
- Provides word index information

**Key Methods:**
- `find()` - Execute search query
- `print_word()` - Get word statistics
- `format_find_output()` - Format search results
- `format_print_output()` - Format word details

#### IndexPersistence (`src/persistence.py`)

**Responsibilities:**
- Saves index to JSON file
- Loads index from JSON file
- Manages index file location

**Key Methods:**
- `save_index()` - Serialize index to disk
- `load_index()` - Load index from disk
- `index_exists()` - Check if index file exists

### Design Decisions

#### Why Inverted Index?
The inverted index is chosen because:
- **Efficiency**: O(1) lookup time for words
- **Space efficiency**: Stores word-to-pages mapping efficiently
- **Scalability**: Handles large numbers of pages
- **Simplicity**: Easy to implement and understand

#### Why JSON for Persistence?
- Human-readable format
- Language-independent
- No external dependencies needed
- Suitable for educational purposes

#### Case-Insensitive Search
- Simplifies user queries
- More intuitive for general search
- Reduces index size slightly

#### AND Query Logic
- Multiple search terms return intersection of pages
- More precise results than OR queries
- Aligns with user expectations (e.g., Google's implicit AND)

## Performance

### Benchmarks

Synthetic benchmark (`python3 scripts/benchmark.py`) on 1,500 generated pages:

| Operation | Time |
|-----------|------|
| Build Index | 0.0781 s |
| Mean Query Latency | 3.575 ms |
| P95 Query Latency | 11.020 ms |
| Mean Ranked Latency | 36.756 ms |
| P95 Ranked Latency | 87.495 ms |

For real crawling, total build time is still dominated by the required 6-second politeness window between HTTP requests.

### Complexity Analysis

- Index build: approximately O(T), where T is total token count across all pages
- Single-word lookup: O(1) average dictionary access plus output size
- AND query intersection: O(sum(posting list sizes)) in the worst case
- Phrase query: O(C * P), where C is candidate pages and P is positional checks
- Ranked retrieval: O(C * Q), where C is candidate pages and Q is query term count

### Scalability

The implementation handles:
- 1,000+ unique words
- 100+ indexed pages
- 50,000+ total word occurrences

## Implementation Details

### Word Tokenization

Text is tokenized into words by:
1. Converting to lowercase
2. Removing punctuation (except hyphens in words)
3. Splitting on whitespace and special characters
4. Filtering empty strings

**Examples:**
- `"Hello, World!"` → `["hello", "world"]`
- `"It's a test"` → `["it", "s", "a", "test"]`
- `"well-known"` → `["well-known"]`

### Politeness Window

The crawler respects a 6-second minimum between successive requests:
- **Why?** Ethical web scraping and server respect
- **How?** Tracks `last_request_time` and sleeps if needed
- **Non-negotiable**: Required by coursework specification

### Inverted Index Storage

Index structure optimized for:
- Fast word lookups
- Efficient storage
- Easy serialization

Example index:
```json
{
    "python": {
        "pages": [
            {
                "url": "https://example.com/1",
                "frequency": 3,
                "positions": [0, 45, 127]
            },
            {
                "url": "https://example.com/2",
                "frequency": 1,
                "positions": [56]
            }
        ]
    }
}
```

## Error Handling

### Network Errors
- Caught and logged at `ERROR` level
- Returns empty content instead of crashing
- Crawler continues with next page

### Malformed HTML
- BeautifulSoup handles invalid HTML gracefully
- Text extraction still works for partial content
- Logged as warning

### File I/O Errors
- Index save/load failures logged
- Graceful fallback to empty index
- User informed of failure

### Invalid User Input
- Empty queries return no results
- Non-existent words handled gracefully
- Clear error messages provided

## Logging

The application uses Python's `logging` module:

- **INFO**: Normal operation (build progress, page crawling)
- **WARNING**: Recoverable issues (malformed HTML, empty pages)
- **ERROR**: Failures that may affect operation

All output goes to console with timestamp and level information.

## Troubleshooting

### Build Command Takes Too Long
- This is normal! The politeness window enforces 6-second delays between requests
- With 50 pages, build takes approximately 5 minutes
- This is ethical web scraping - necessary and good practice

### Index File Not Found
- Run `build` command first to create the index
- Check that `data/` directory exists and is writable

### Search Returns No Results
- Verify word exists in indexed content
- Search is case-insensitive, so capitalization doesn't matter
- For multi-word searches, all words must appear in same page

### Import Errors
- Verify all files in `src/` directory are present
- Check that Python version is 3.7+
- Ensure `requests` and `beautifulsoup4` are installed
